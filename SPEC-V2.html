Product overview
Funnel Messaging Consistency & Analytics Dashboard
A two-tab analytics tool that helps performance marketers verify messaging alignment across the full acquisition funnel — from paid ad to product onboarding — and measure quantitative performance at each stage. The primary axis of analysis is always Audience × Channel × Selling Point.

Tab 1 — qualitative: messaging consistency
Purpose
For a selected Audience + Channel + Selling Point combination, show the marketer the actual creative content at each funnel stage side-by-side, so they can visually verify that the selling point introduced in the ad is carried through to the landing page and into the onboarding experience.

Consistency score logic
Selling point consistency score (per row)
For each active selling point in the selected ad, compute a binary or 3-level presence score at each downstream stage: Present above fold Present below fold Absent. Roll up into a single consistency indicator per funnel path. This score is displayed both in the 3-stage view and as a sortable column in the row-level table below.
Row-level table beneath the side-by-side view
A filterable table where each row = one unique Audience + Channel + Selling Point + Ad variant combination. Columns: Audience, Channel, Selling point, Ad variant ID, LP variant ID, Onboarding variant ID, Ad→LP consistency, LP→Onboarding consistency, Overall consistency score, Last updated.
Data requirements for this tab
Ad taxonomy / creative library
Each ad must be tagged with: audience segment, channel, selling point(s), creative asset URL, headline, body copy, CTA. Source: ad platform export or internal creative CMS.
Landing page content map
Each LP variant tagged with its UTM parameters and a structured content schema: hero headline, subheadline, CTA, value prop list with fold position. Source: CMS or manual content tagging layer.
Onboarding content map
Each onboarding flow variant tagged with: welcome copy, per-step headlines and feature highlights, selling point tags. Source: product team export or in-app content layer.
Audience → variant mapping
A lookup table linking Audience segment + Channel to the specific Ad variant, LP variant, and Onboarding variant they received. This is the spine that joins all three stages.

Day-2 return rate from each ad → LP path
For users who signed up via a specific Ad + LP path, track whether they return to the product on Day 2 (defined as a session starting between 24h and 48h after first signup session). Data model — entities and relationships
AudienceSegment
id, name, channel, targeting_type (KW / interest / retargeting / lookalike), keyword (nullable for non-search)
AdVariant
id, audience_segment_id, channel, headline, body_copy, cta_label, asset_url, selling_points[ ], impressions, clicks, ctr
LandingPageVariant
id, utm_params, hero_headline, subheadline, cta_label, value_props[ {text, fold_position} ], selling_points[ ]
LPSession
id, lp_variant_id, audience_segment_id, session_duration_s, scroll_depth_pct, bounced, cta_clicked, converted, timestamp
OnboardingVariant
id, steps[ {headline, copy, selling_points[ ]} ], welcome_copy, selling_points[ ]
UserOnboardingEvent
user_id, ad_variant_id, lp_variant_id, onboarding_variant_id, signup_ts, day2_returned (bool), future_events[ ]
FunnelPath
The join key: audience_segment_id → ad_variant_id → lp_variant_id → onboarding_variant_id. Every metric in both tabs resolves back to this 4-way mapping.
SellingPoint (enum/taxonomy)
A shared controlled vocabulary of selling points (e.g. Speed, Price, Ease of use, Social proof, Security) used consistently across Ad, LP, and Onboarding tagging to enable cross-stage comparison.
Key interactions and cross-tab links
From quantitative → qualitative drill-down
Clicking a row in the funnel breakdown table opens the qualitative 3-stage view pre-filtered to that exact Audience + Channel + KW + Selling Point. This is the core diagnostic workflow: spot an anomaly in the numbers, then inspect the creative to understand why.
Consistency score as a filter in the quant tab
The user can filter the quantitative table to only show rows where consistency score is "partial" or "absent" to prioritize investigation.
Selling point presence toggle in qualitative view
Within the 3-stage side-by-side, highlight/dim content blocks based on which selling point is selected, so the marketer can visually trace a single message through the funnel.
Edge cases and constraints agents should handle
Multiple selling points per ad
An ad may carry more than one selling point. The consistency check runs independently for each. The overall consistency score for the path is the mean or minimum across all active selling points — product owner to decide aggregation method.
Missing UTM linkage
If LP sessions cannot be attributed back to a specific ad/audience (missing or broken UTMs), surface a data quality warning on the affected rows in the quant tab rather than silently dropping the data.
Search vs. social channel differences
Search ads use keywords as the primary segment key; Meta and YouTube use audience/ad set. The data model and filters must accommodate both without forcing a false equivalence.
Onboarding variant not yet defined
If a funnel path has no mapped onboarding variant (e.g. single universal flow), Stage 3 in the qualitative view shows the universal flow with a note that it is not audience-specific.
